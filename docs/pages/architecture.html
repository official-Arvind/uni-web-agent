Architecture Overview

The Jigar Universal Web Agent implements a Hybrid Cache architecture.

1. The Mapping Phase (AI)

Using Crawl4AI to flatten shadow DOMs and trafilatura to clean the HTML, the page is sent to Gemini 3.5 Flash. Gemini returns a robust, deterministic workflow config in JSON format.

2. The Execution Phase (Zero-Token)

The Python FastAPI backend reads the workflow config and executes it via Playwright and Camoufox. Zero AI calls are made during normal execution, achieving 100% cost reduction per run.

3. The Auto-Healing Phase

If a selector fails, the engine captures a snapshot of the Accessibility Tree and falls back to Gemini 3.5 Flash to identify the new selector. The config is updated, and execution resumes.
